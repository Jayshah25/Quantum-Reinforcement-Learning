# 🚀 qrl-qai v1.0.0

We're thrilled to announce the first major release of qrl-qai. This release marks a significant milestone — expanding the SDK beyond quantum environments into classical tabular RL algorithms, and introducing the first environment designed specifically to work with them.

---

# ✨ What's New

## 1. New Environment — BlochSphereV1

A discrete, graph-based reformulation of the Bloch sphere environment.

- The qubit state space is discretized into six canonical pure states: `|0⟩`, `|1⟩`, `|+⟩`, `|-⟩`, `|+i⟩`, `|-i⟩`, exposed to the agent as an integer index.
- The action space is reduced to four deterministic gate actions — **H, X, Z, S** — forming a clean finite MDP over the canonical Bloch-sphere axes.
- Reward is **binary and sparse**: `1.0` on reaching the target state, `0.0` otherwise.
- Fully compatible with the modern Gymnasium API, returning a 5-tuple `(obs, reward, terminated, truncated, info)` from `step()`.
- The first qrl-qai environment natively compatible with tabular planning algorithms without any wrapper.
- Includes a **2D graph-based renderer** with two panels:
  - *Left panel*: True environment dynamics with current state, target state, and episode trajectory highlighted.
  - *Right panel*: Agent's learned model — nodes coloured by V(s) or max_a Q(s,a) on a warm colormap, edge opacity proportional to visit counts, and bold teal edges marking the greedy policy.

## 2. New Algorithms — ValueIteration and QValueIteration

The first classical RL algorithms shipped with qrl-qai, laying the foundation for the algorithm side of the SDK.

**ValueIteration**
- Maintains a state-value function V(s) and applies the Bellman optimality operator until convergence.
- Action selection uses a one-step lookahead over the empirical transition and reward model.
- Convergence is detected via the sup-norm `|V_new - V|∞ < tol`.

**QValueIteration**
- Maintains a state-action value function Q(s,a) directly.
- Action selection is a direct O(n_actions) table lookup — no recomputation at inference time.
- V(s) = max_a Q(s,a) is available as a derived property.
- The natural precursor to Q-learning and function approximation methods such as DQN and quantum RL agents.

Both algorithms:
- Are implemented in **PyTorch**, with support for CPU and CUDA devices.
- Build an **empirical model** of the environment from interaction via `play_n_random_steps()`, requiring no access to ground-truth transition dynamics.
- Are interoperable with any `gymnasium.Env` that has discrete observation and action spaces — including all qrl-qai environments.
- Share a common `BaseIteration` base class that manages transition counts, reward accumulation, and model update logic.

# 📦 Highlights

- `BlochSphereV1` + `ValueIteration` / `QValueIteration` form the first complete **plan → evaluate → visualize** workflow in qrl-qai.
- Both planning algorithms ship with a `get_policy()` method returning the full greedy policy as a tensor, enabling direct inspection and downstream use.
- The `BaseIteration` class is designed to be subclassed — future quantum planning algorithms can inherit the empirical model machinery and override only the Bellman update.
